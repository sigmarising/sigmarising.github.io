## 无线电手台

观察到很有意思的一个现象，真正考了证去使用业余无线电台的 HAM，大部分都不会购买小米的手台；而大量不知道还需要考证的人，购买小米的手台还蛮多的。

## RAV4

我的 RAV4 驾驶了也将近一年多的时间了，虽然次数不多，但是几乎每次都是长途。关于这辆车的驾驶感受我觉得还是有发言权稍微聊一聊的。我的 RAV4 是 2024 款风尚 Plus（即两驱汽油顶配版），经过简单的玻璃贴膜、ETC 安装、前后拉行车记录仪安装后，再无其他配件了。

先说说缺点部分吧，相对比较零散：
- 首先就是纸质手册是不完整版本的，且其声称完整版请通过一汽丰田 APP 查看。但可笑的是这个垃圾一丰的 APP 电子手册我根本找不到对应我车型号版本的，只能好几个手册一起看着互补，有点离谱
- 驾驶位车窗控制部分，很多键是没有夜光的，例如门锁按键，只能靠按键上的小凸起盲摸
- 360 全景影像，我只能说能用，和新能源没法比；但好在其响应迅速（独立于车机系统）且从未出 BUG
- 车机，RAV4 国产版本是外包给 HW 做的，车机易用性真的一言难尽，且发动机点火后启动速度极慢……但好在能用无线 CarPlay，我也只用 CarPlay（就是有较低概率在某次发动机点火后连不上手机，需手动重连；但好在 CarPlay 只要连上了就没有出现过断连情况）
- 音响，凑合能听，低频比较烂；同时整车隔音也比较一般
- LDA 车道居中效果一般，在我个人预期之下，但**好在我不会过度依赖于 L2 辅助驾驶**
- 2.0 汽油版本的动力真的肉，比我妈的同款发动机弈泽肉太多了……可能 2.5 油混版本会好很多，但又有国六 B + 颗粒捕捉器这种精准打击丰田的阳谋……

至于说其他方面，我真的挑不出什么大毛病了，RAV4 就是一款纯粹的将我从 A 运送到 B 的工具车。保养简单，纯油车也无需担心颗粒捕捉器堵塞问题，仅需 92 号汽油就可以喂饱，大量实体按键，储物空间够用，（凑合能用的）360全景 + 前排座椅加热、通风、电动控制 + 电动后尾门一个不少。对于我这种一年开不了特别多次，但每次基本都是长途的，RAV4 完美契合了我的个人需求。

但话又说回来了，你但凡对汽车稍微有点某方面的追求，就不应该选择丰田。但恰好我对于汽车的确没啥追求，对于将我从 A 安全的运送到 B 而言，RAV4 足矣。

## MacBook Air

MBA 是我在今年 618 时，因受到四月某个事件影响后，一狠心购入的（M4 蓝色 32G + 1T）。在刚购入的前三个月内，我还是有一点后悔的（使用频次极低），但在经历了九月某个事件后（使用频次急速上升），直到年底，我一点后悔之意也没有了，反而庆幸当时狠下心来入手 MBA 了。

MBA 最大的优点就是颜值好看、做工精致、小巧便携（相比于蠢厚的 MBP）。其实这些都没得说了，毕竟价格摆在这里了，它的确比我用过的任意一台 Windows 笔记本做工都要好。在 Windows 阵营，唯一能和 MacBook 掰掰手腕的 XPS 如今已经寄了，那就按下不谈了。至于 MacBook 无法 180 度展开、明确说明屏幕惧怕酒精，那都不是什么特别关键的问题了。

P.S. 作为少数不吃国补涨价的厂商，这点真的要点个赞

说句实话，这并非是我第一次使用 Mac，在曾经两家互联网公司实习时，均是使用的 MacBook Pro，在系统适应上于我而言不成问题。但还是有一些部分不吐为快：
- 触控板仅限轻度使用，用得久我还是习惯于用鼠标（Logitech Master 3），毕竟怕得腱鞘炎（且按压拖拽经常性误触失效）
- 液态玻璃过于扯淡，又难看、又吃性能、又费电、又可读性差；我的 MBA 系统已经停留于 MacOS 15.7 了……
- 安装/卸载软件十分方便，但无奈仅靠倾倒废纸篓卸载软件是卸不干净的（甚至 Microsoft Office 都有一个专门的卸载文档）
- 基于安全芯片的抹掉助理十分便利，虽然我依旧怀疑其是否真的抹的干净，参考：
  - [苹果 iOS / iPadOS 17.5 “照片复活门”升级：用户反馈二手设备显示原机主照片](https://www.ithome.com/0/768/883.htm)
  - [苹果 iOS 17.5.1 正式版发布：修复已删除照片“复活”问题](https://www.ithome.com/0/769/417.htm)
- Finder 除了分栏和列表模式好用之外，其他视图模式根本没法用
  - 离谱的是 iWork 套件都支持 CMD + X 指令，而 Finder 中只能 OPT + CMD + V，这就是果蛆口中的 Apple 一致和优雅性？
- 音响效果比 Win 笔记本强，但是依旧强不过贴牌的飞利浦条形桌面音响（音响体积差距面前，众生平等）
- 少部分软件，Mac 版不如 Win 版好用（例如 Snipaste 和 BandiZip）
- **由于 Mac App Store 的沙箱机制，商店版应用往往功能和稳定性都严重逊色于官网下载版本**（例如达芬奇、Parallels Desktop）

本人是 BUPT 的 CS 专业本硕科班出身，但我并没有觉得 Mac 相比 Win 在开发上有什么绝对碾压性质的银弹级别优势。反而由于 Windows 的 WSL2 的缘故，绝大部分情况是能在 Mac 上开发的，Win 上基本也可以，但是反过来可能不行（这里均抛开 Mac/iOS 开发以及 Windows 开发）。

Mac 俗话所说的 Code 友好，其实是建立在熟练之上的（例如命令行），但是熟练者使用 Windows 依然不会太差；同理，对于新手而言，无论是 Win 还是 Mac 他们都会头疼不已……

而 Mac 真正相比 Win 具备的绝对优势，在我看来主要有三点：
1. 高效的音视频编解码（MIDI 音频创作 + 视频创作）
2. HDR 管理
3. 离电性能和续航

其中前两点主要优势在省心省力上；而最后一点，就微软现在的 Bug 管理水平，我感觉 Windows 至少十年都追不上 Mac 了（甚至微软默认把“休眠”按钮是藏起来的，搞不明白其蜜汁自信在哪里）……

所以综合来看，Mac 真的是非常适合自媒体工作者的生产力神机了。但话又说回来，其实在国内，基于各种各样的现实因素，大部分人的第一台电脑真的并不适合选 Mac。而我也是在有了高性能 Windows 台式机 + 各种各样的 Windows 笔记本后，才选择的 MBA 作为离电轻量级别创作电脑。

Mac 并非神器，它也有 Bug，且 Bug 不一定比 Windows 少……下面给出几条 Mac 的使用小建议：
- 对于重度使用的生产力软件，不要使用 Mac App Store 版本，应该选用官网下载版（例如专业视频创作者使用达芬奇）
- 对于轻度使用（或有些流氓）的软件，可以选用 Mac App Store 版本轻量使用（例如业余爱好者使用达芬奇；WPS 以及 Microsoft 365 捆绑安装套件）
  - iWork 办公套件大部分时间都是自己玩玩用的，毕竟国内是 WPS/Office 打天下，国外是 Google G-Suite 打天下……
- 能使用 MacOS 自带功能解决的，尽量不要严重依赖第三方，除非第一方软件实在太不好用（例如终端和压缩软件）
- 尽可能不要依赖任何的第三方“优化”软件，参考：
  - [请卸载腾讯柠檬清理，清理垃圾之后很多软件已损坏，打不开](https://www.zhihu.com/question/391080330/answer/33988557187)
- MacOS 不像 iOS，很多内置 App 是无法卸载的，但也请不要使用任何 Hack 手段尝试卸载任何内置 App
